# Docker Compose configs collection

# Excluded - Empty Directories

Services present in the Docker Compose file tracked as empty - excluded bind mounts directories from version control outlines some files and folders are by default created - requested by the container mount to work as expected (e.g., backups, cache, logs, configs) so it may contain large or system-generated content, corresponds to a per-individual set-up basis or/and it may differ among other similar settings. 

On the other hand, tracked - non empty bind mount directories on version control carries on some special settings and configs that need to be highlighted.
